# AI Usage Log# AI document describes how AI tools (ChatGPT / Microsoft Copilot) were used during the development of the **Campus Maintenance & Complaint Management System**, in accordance with academic integrity and transparency guidelines.

---

## 1. Purpose of AI Usage

AI assistance was used **as a learning and productivity aid**, not as a replacement for understanding or independent problem‑solving. The primary goals were:

- Understanding full‑stack and microservice architecture concepts
- Accelerating boilerplate code creation
- Debugging errors and resolving configuration issues
- Improving code structure, UI consistency, and documentation quality

All final decisions, integrations, testing, and submissions were performed by the student.

---

## 2. Areas Where AI Was Used

### 2.1 Backend Development

AI assistance was used to:
- Understand JWT‑based authentication and authorization flow
- Structure Express.js route files and middleware
- Design role‑based access control (Student / Admin / Maintenance)
- Debug HTTP errors such as 401, 403, and CORS issues
- Clarify and enforce the complaint lifecycle (OPEN → IN_PROGRESS → RESOLVED)

**Student responsibility:**
- Writing and integrating backend code
- Connecting and managing MongoDB
- Verifying API behavior using Postman and Swagger

---

### 2.2 Frontend Development

AI assistance was used to:
- Design React component structure
- Handle form state and API integration using Axios
- Implement role‑based navigation and conditional rendering
- Improve UI consistency using reusable components (NavBar, PageContainer)
- Debug React rendering and state‑management issues

**Student responsibility:**
- Integrating frontend components
- UI styling and layout decisions
- Ensuring correct data flow between frontend and backend

---

### 2.3 Debugging & Troubleshooting

AI was used extensively for:
- Diagnosing backend and frontend runtime errors
- Understanding and fixing Swagger configuration issues
- Resolving central Swagger aggregation challenges
- Identifying and correcting logical bugs in UI behavior

All suggested fixes were validated manually by the student.

---

### 2.4 API Documentation (Swagger)

AI assistance was used to:
- Learn Swagger (OpenAPI 3.0) syntax
- Write Swagger JSDoc annotations for all backend APIs
- Configure Swagger UI for individual services
- Implement a centralized Swagger documentation approach using JSON merging

**Student responsibility:**
- Integrating Swagger into each service
- Verifying accuracy of API documentation
- Testing Swagger endpoints manually

---

### 2.5 Documentation & README

AI assistance was used to:
- Structure project documentation
- Improve clarity, grammar, and formatting of README.md
- Generate professional‑quality explanations for system architecture and workflows

All documentation was reviewed, edited, and finalized by the student.

---

## 3. Nature of AI Contributions

AI contributions were limited to:
- Conceptual explanations
- Code suggestions and examples
- Error analysis and debugging guidance
- Documentation drafts

AI did **not**:
- Automatically submit code
- Deploy the application
- Perform testing
- Make architectural decisions independently

---

## 4. Academic Integrity Statement

The student confirms that:
- AI tools were used responsibly as support tools
- All submitted work was understood and verified
- No direct copying was done without comprehension
- The project reflects the student’s own learning and implementation

---

## 5. Summary

AI tools significantly improved productivity and learning efficiency, particularly in debugging, documentation, and best‑practice guidance. However, the overall system design, coding, testing, and final submission were completed independently by the student.
